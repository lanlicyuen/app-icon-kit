# Windows应用图标生成指南

## 📱 App Logo图片要求

### **推荐规格**
- **尺寸**：1024x1024像素（正方形）
- **格式**：PNG（推荐，支持透明背景）
- **分辨率**：300 DPI或更高
- **文件大小**：建议小于5MB

### **设计要求**
- **简洁明了**：避免复杂细节，确保在小尺寸下也能清晰识别
- **高对比度**：确保在各种背景下都清晰可见
- **无文字**：图标中不建议包含文字，因为小尺寸下无法识别
- **圆角适配**：考虑不同平台的圆角处理

## 🪟 Windows图标生成功能

### **支持的尺寸**
| 尺寸 | 用途 |
|------|------|
| 16x16 | 任务栏小图标 |
| 32x32 | 桌面图标、资源管理器 |
| 48x48 | 控制面板 |
| 64x64 | 高DPI显示 |
| 128x128 | 大图标视图 |
| 256x256 | 超高分辨率显示 |

### **生成的文件**
1. **PNG文件**：所有尺寸的独立PNG文件
2. **ICO文件**：包含所有尺寸的Windows图标文件

### **使用方法**
1. 启动 `iOS & Android & Windows 图标生成器`
2. 选择源图片（1024x1024正方形图片最佳）
3. 勾选 "Windows" 平台
4. 设置样式（可选）
5. 选择输出目录
6. 点击 "生成图标"

## 📂 输出文件结构

```
输出目录/
├── Windows/
│   ├── Small (16x16)_16x16.png
│   ├── Medium (32x32)_32x32.png
│   ├── Large (48x48)_48x48.png

│   ├── Extra Huge (512x512)_512x512.png
│   └── app_icon.ico  (多尺寸ICO文件)
```

## 💡 最佳实践

### **设计建议**
1. **使用矢量图设计**：确保在任何尺寸下都清晰
2. **测试小尺寸**：确保16x16像素下仍可识别
3. **避免渐变**：小尺寸下渐变效果不佳
4. **使用鲜明颜色**：提高识别度

### **技术建议**
1. **检查透明度**：确保PNG透明区域正确
2. **验证ICO文件**：在不同Windows版本测试
3. **备份原图**：保留原始高分辨率图片

## 🚀 集成到项目

### **WPF应用程序**
```xml
<Window Icon="app_icon.ico">
```

### **Windows Forms**
```csharp
this.Icon = new Icon("app_icon.ico");
```

### **Electron应用**
```javascript
mainWindow.setIcon('path/to/app_icon.ico');
```

## 📋 注意事项

1. **版权问题**：确保使用的图片有合法版权
2. **平台规范**：遵循Windows设计规范
